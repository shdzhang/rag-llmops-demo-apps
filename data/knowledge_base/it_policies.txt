# IT Policies and Support

## Acceptable Use Policy

### General Principles

Company IT resources are provided for business purposes. Limited personal use is permitted as long as it:
- Does not interfere with work duties
- Does not compromise security
- Complies with all company policies

### Prohibited Activities

The following are strictly prohibited:
- Installing unauthorized software
- Sharing passwords or credentials
- Accessing inappropriate content
- Using company resources for personal business ventures
- Downloading pirated software or media
- Circumventing security controls
- Using personal devices to access confidential data (unless MDM enrolled)

Violations may result in disciplinary action, up to and including termination.

## Password Policy

### Requirements

All passwords must:
- Be at least 12 characters long
- Include uppercase, lowercase, numbers, and special characters
- Not contain personal information (name, birthdate, etc.)
- Not reuse any of last 10 passwords
- Be changed every 90 days

### Multi-Factor Authentication (MFA)

MFA is required for:
- Email and calendar access
- VPN connections
- Cloud services (AWS, Azure, GCP)
- Source code repositories
- HR and finance systems

Use Microsoft Authenticator or Google Authenticator app for MFA tokens.

### Password Manager

We provide 1Password Business for all employees:
- Store all work passwords in 1Password
- Never write down or share passwords
- Use strong, unique passwords for each system
- Enable emergency access for your manager

## Device Management

### Company-Issued Devices

**Laptops**
- Standard issue: MacBook Pro (M3) or Dell XPS 15
- Refresh cycle: Every 3 years
- Encryption: FileVault (Mac) or BitLocker (Windows) required
- Antivirus: Company-managed endpoint protection

**Mobile Devices**
- Optional: iPhone or Android phone with MDM enrollment
- Monthly stipend: $100 for BYOD (Bring Your Own Device)
- MDM required for email access

### Personal Devices (BYOD)

Personal devices can access company resources if:
- Enrolled in Mobile Device Management (MDM)
- Passcode/biometric lock enabled
- Company can remote wipe business data if device is lost

BYOD devices cannot access:
- Highly confidential data (classified as "Restricted")
- Production systems
- Source code repositories

### Lost or Stolen Devices

**Immediate Actions:**
1. Report to IT Security immediately: security@company.com or call 1-800-XXX-XXXX
2. Remote wipe will be initiated within 1 hour
3. File police report if stolen
4. Replacement device shipped within 2 business days

## VPN Access

### When to Use VPN

VPN is required when:
- Working from home or remote location
- Using public WiFi (coffee shops, airports, hotels)
- Accessing internal systems not exposed to internet

VPN is NOT required for:
- Office network
- SaaS applications (email, Slack, Salesforce)

### VPN Setup

1. Download Cisco AnyConnect from IT Portal
2. Connect using your email address and password
3. Approve MFA prompt
4. Connected! Green lock icon indicates secure connection

### VPN Troubleshooting

**Cannot connect?**
- Verify internet connection
- Check credentials (password may have expired)
- Restart VPN client
- Contact IT support if issue persists

## Data Classification and Handling

### Classification Levels

**Public**
- Marketing materials
- Published blog posts
- Job postings
- Can be shared freely

**Internal**
- Business documents
- Internal presentations
- Project plans
- Share only with employees

**Confidential**
- Customer data
- Financial records
- Unreleased product info
- Share only on need-to-know basis

**Restricted**
- Trade secrets
- Legal documents
- M&A information
- Executive-level only, special approval required

### Data Protection

**Email**
- Encrypt emails with confidential attachments (add [SECURE] to subject)
- Do not forward confidential info to personal email
- Verify recipient before sending sensitive data

**File Sharing**
- Use company-approved tools: Box, SharePoint, Google Drive
- Do not use personal Dropbox, WeTransfer, etc.
- Set appropriate permissions (view-only vs. edit)
- Set expiration dates for external shares

**Printing**
- Minimize printing of confidential documents
- Use secure print release (badge required at printer)
- Shred confidential documents after use

## Cloud Services Access

### AWS Access

Request via IT Service Portal:
- **Developer Access**: Read-only for dev/test environments
- **Power User**: Deploy access for staging environments
- **Admin Access**: Full access for production (requires manager + InfoSec approval)

Access is reviewed quarterly. Inactive accounts are disabled after 30 days.

### Azure Access

Similar process to AWS. Subscriptions are organized by:
- Development
- Staging
- Production

Production access requires MFA + bastion host.

### GCP Access

Request project access via IT Portal. Default IAM roles:
- Viewer (read-only)
- Editor (deploy and modify)
- Owner (full admin, requires approval)

## Remote Work IT Setup

### Required Equipment
- Company laptop (encrypted)
- VPN client installed
- MFA app configured
- External monitor (optional, can request via IT Portal)

### Home Network Security
- Use WPA2/WPA3 WiFi encryption
- Change default router password
- Enable router firewall
- Keep router firmware updated

### Troubleshooting Common Issues

**Slow VPN Connection?**
- Restart router
- Close bandwidth-heavy apps (streaming, large downloads)
- Switch to wired ethernet connection

**Cannot access internal tools?**
- Verify VPN is connected
- Clear browser cache
- Try incognito/private browsing mode

**Laptop won't start?**
- Hold power button for 10 seconds to force restart
- If issue persists, contact IT for replacement
